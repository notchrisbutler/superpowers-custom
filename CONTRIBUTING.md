# Contributing

Thanks for helping improve SuperDuperPowers.

---

## Before Opening An Issue

- Search existing issues first.
- Include the harness name and version when reporting behavior.
- Include the SuperDuperPowers version from `package.json` or `CHANGELOG.md`.
- Keep reports focused on one bug, feature, or harness concern.

---

## Pull Requests

Pull requests should include:

- The problem being solved
- A concise summary of the change
- Verification performed
- Harness compatibility risks for OpenCode, Claude, Codex, Cursor, Gemini, or Copilot when relevant

---

## Skill Changes

Skills shape agent behavior. Keep edits direct, operational, and tested with realistic prompts when practical.

Avoid adding process unless it solves a concrete problem for this project.
